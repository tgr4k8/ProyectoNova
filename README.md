# ProyectoNova
Aplicación web para academia de baile NOVACLUB. Gestor de contenido ERP creado por mi para el sitio con posibilidad de añadir, clases, estilos, grupos y eventos. Diferentes tipos de usuarios gestionados por JWT, diferenciando entre: Suscrito, Invitado, Profesor y Administrador diferenciando funcionalidades por roles.

# Funcionalidades del sitio:
1. Login y registro de usuarios. (En proceso validación de usuario por mail)
2. Página de inicio con carrusel para los estilos y los profesores que imparten clases en el sitio.
3. Página de grupos de dificultad con animacion tipo orbitas para la selección del grupo de baile (planetas).
4. Página de estilos con los estilos asiganadas al mismo grupo.
5. Página de clases con las clases asignadas al mismo estilo y mismo grupo.
6. Página de clase con dos posibles vistas de video por clase, una delantera y otra trasera, sincronizadas entre si: si una se para la otra también, si una se reproduce la otra también parandose el sonido de una de ellas para evitar sonidos sucios.
7. Calendario personalizado para el sitio con posibilidad de que el administrador pueda incorporar eventos externos para reuniones o eventos entre los alumnos y crear comunidad.
8. Página de comunidad con sistema de puntos "Novanitos" con rankings de posición incentivando el game-site y crear comunidad.
9. Foro de comunidad para preguntar dudas o sugerir contenido a los administradores. (En proceso).
10. Portal para administradores tipo ERP para añadir, editar o eliminar clases, estilos, grupos, eventos y usuarios.
11. Sistema de pagos mediante Stripe. Página de tarifas con diferentes tipos de suscripción autogestionado para cambiar de tipo de usuario en cuanto acaba la suscripio cambio automatico de tipo de usuario a invitado. (En proceso).
12. Pagina de usuario donde puede añadir imagen de usuario, asi como poder ver sus clases completadas.
13. Diseño integrado de un diseño de UX/UI creado en figma, diseño responsive.
14. Gestión de cookies.
15. Metodos de gestión para evitar la multicuenta y fraude. (En proceso).


# Tecnologias utilizadas:
Gestión de tiempos, tareas y control de incidencias mediante Trello.
Comunicación con cliente y UI/UX mediante meets de google y grupo de Whatsapp.
Gestion de diseño mediante Figma.

Node.js
# Frontend:
